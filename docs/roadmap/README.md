# Roadmap y features

Acá vive el **qué construir** y en qué orden.  
El diseño detallado de cada sistema sigue en `gameplay/`, `content/`, etc.

| Doc | Para qué |
|-----|----------|
| [mvp.md](./mvp.md) | Checklist de lo que entra en la primera versión jugable |
| [features.md](./features.md) | Catálogo de features con estado |
| [backlog.md](./backlog.md) | Próximas features (post-MVP) priorizadas |

## Estados de una feature

| Estado | Significado |
|--------|-------------|
| `idea` | Apuntada, sin compromiso |
| `mvp` | Obligatoria para la primera versión jugable |
| `next` | Justo después del MVP |
| `later` | Deseable, no priorizada |
| `done` | Hecha y verificable en juego |
| `cut` | Descartada a propósito |

## Regla

- Si algo está en **MVP**, tiene que poder tildarse con un criterio claro de “listo”.
- El detalle de diseño no va acá: linkeá al doc de `gameplay/` o `content/`.
- Cambios de alcance del MVP se anotan en [mvp.md](./mvp.md) y, si son grandes, en un ADR.
